## Critical Integration Issues

### Bug 1: Scheduler Argument Order Inversion (CRITICAL)

**File:** `job_finder/web/scheduler.py` lines 146, 393
**Affected:** REFAC-06, SAFE-03

`run_ingestion` signature is `(config: dict, db_path: str)` but both scheduler closures call:
```python
summary = run_ingestion(db_path, config)  # WRONG — args swapped
```
Result: `config.get("sources", ...)` is called on a string path → `AttributeError` every invocation. Scheduled ingestion and "Sync Now" never succeed.

**Why tests don't catch it:** `test_scheduler.py` patches `run_ingestion` entirely; `test_ingestion.py` calls it with correct arg order.

### Bug 2: ScoringResult NamedTuple Treated as Dict (CRITICAL)

**File:** `job_finder/web/pipeline_runner.py` lines 509-518, 651-660
**Affected:** CLN-02, REFAC-04

Phase 8 changed scorers to return `ScoringResult` (NamedTuple). `pipeline_runner._run_haiku_scoring` still does:
```python
result = score_job_haiku(...)
score = result.get("score", 0)  # AttributeError: NamedTuple has no .get()
```
Same pattern in `_run_sonnet_evaluation`. The `scoring_orchestrator` correctly handles `ScoringResult` but `pipeline_runner` bypasses it.

**Why tests don't catch it:** `test_data_enricher.py` mocks scorer to return plain dict; `test_ingestion.py` patches `anthropic` to None (skips scoring).
