## Methodology

### Baseline sampling
- Total eligible Anthropic-filtered pool size: **623** rows
- Sampled n=100 (80 dev + 20 holdout), stratified across 4 score quartiles (25 per bucket)
- Filter SQL: `jobs.scoring_provider='anthropic' AND sonnet_score IS NOT NULL AND LENGTH(TRIM(jd_full)) >= 200 AND EXISTS (SELECT 1 FROM scoring_costs sc WHERE sc.job_id=j.dedup_key AND sc.provider='anthropic' AND sc.purpose IN ('sonnet_eval','haiku_score'))`

### Gold baseline
- Model: `claude-opus-4-6`
- Prompt sha256: `255c690e06ee58c87d32dc19ef4abd8ca25e9339eae009a327762f6de2d0c9da`
- Prompt source: Plan 1 commit `171e41d`
- Cumulative Opus spend: $0.0000
- Hard budget cap: $30.00 (D-14)

### Statistical methods
- Stat method: paired per-dim MAE (6 ordinal axes) + BCa bootstrap 10k resamples, 95% CI, random_state=42 per D-15/D-16/D-17
- No significance testing; point estimates + CIs only (D-18)

### Gate definitions
- Schema-retry rate threshold: `> 0.2` → WARN (suppressed when n < 20)
- Determinism probe: 5× byte-identical output on 3 scoring fixtures (scoring sites only per D-19)
- VRAM reset between candidates: `ollama stop` + poll `nvidia-smi` until memory.used < 1000 MB
- Gate failure behavior: flag-and-continue (D-21); no auto-exclusion

### Excluded candidates
- `qwen3.5:27b` — Broken Ollama package — community port (family=qwen35, not in official Ollama library), 13-char chat template, returns empty response body with eval_count>0 across all prompts. Confirmed via raw /api/generate on a clean GPU.
- `qwen3.5:14b` — Same broken qwen35 community-port lineage as 27b — excluded by inference; not retested.
- `gemma4:26b-moe` — Ollama structured-output bug (issue #15260) blocks JSON schema path
