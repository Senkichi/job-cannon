<tasks>
<task type="execute">
<action>
Create evals/cascade_audit/judge.py with judge protocol:

1. Define JUDGE_SYSTEM_PROMPT constant:
   - "You are an expert evaluator for job search automation systems. Compare two LLM outputs (A and B) for the same input."
   - Evaluation criteria (in order of priority):
     - Functional correctness: Does the output contain all required fields?
     - Semantic accuracy: Is the extracted information factually correct?
     - Completeness: Does the output include all relevant information?
   - "If both outputs are equally good, return 'tie'. If one output has a critical error (missing required field, invalid URL, hallucinated data), the other wins."
   - "Provide a brief rationale citing specific differences."

2. Define judge_pair function:
   - Accept: output_a (dict), output_b (dict), callsite (str), provider (OpenRouterProvider)
   - Build prompt with callsite context and anonymized outputs labeled A and B
   - Call provider.call() with:
     - model="deepseek/deepseek-chat:free"
     - system=JUDGE_SYSTEM_PROMPT
     - messages=[{"role": "user", "content": prompt}]
     - output_schema=Verdict.model_json_schema()
     - temperature=0 (deterministic)
     - max_tokens=1024
   - Parse response.data into Verdict using Verdict.model_validate_json()
   - Add retry logic: if ValidationError, retry once with same inputs
   - Return Verdict object

3. Define judge_with_position_swap function:
   - Accept: output_a (dict), output_b (dict), callsite (str), provider (OpenRouterProvider)
   - Call judge_pair(output_a, output_b, callsite, provider) → verdict_ab
   - Call judge_pair(output_b, output_a, callsite, provider) → verdict_ba
   - Compute agreement: verdict_ab.winner == verdict_ba.winner or both tie
   - If agreement: return verdict_ab, True
   - If disagreement: return Verdict(winner="tie", rationale="Position swap disagreement", confidence=0.5), False

4. Add docstrings and type hints for all functions
</action>
