# Plan 01: OpenRouter Provider Adapter

<objective>
Create a full BaseProvider implementation for OpenRouter that integrates with the existing provider architecture, enabling DeepSeek-V3.2 judge calls via the :free tier endpoint.
</objective>

<read_first>
- job_finder/web/providers/base_provider.py

</read_first>

<tasks>
<task type="execute">
<action>
Create job_finder/web/providers/openrouter_provider.py with OpenRouterProvider class implementing BaseProvider:

1. Define OpenRouterProvider class inheriting from BaseProvider
2. __init__ method:
   - Accept config dict
   - Load OPENROUTER_API_KEY from environment variable
   - Raise ValueError if API key not set
   - Set _base_url to "https://openrouter.ai/api/v1"
3. call method with signature:
   - model: str
   - system: str
   - messages: list[dict]
   - output_schema: dict | None = None
   - max_tokens: int = 1024
   - timeout: float | None = None
4. HTTP POST to /api/v1/chat/completions:
   - Headers: Authorization: Bearer $OPENROUTER_API_KEY, Content-Type: application/json
   - Body: model, messages, temperature=0 (for judge consistency), max_tokens
   - If output_schema provided, add response_format with type json_object
5. Parse response:
   - Extract content from choices[0].message.content
   - Parse JSON content
   - Extract input_tokens and output_tokens from usage
6. Return ModelResult with:
   - data: parsed JSON
   - cost_usd: 0.0 (free tier)
   - input_tokens: from usage
   - output_tokens: from usage
   - model: model parameter
   - provider: "openrouter"
   - schema_valid: True (assume valid for judge output)
</action>
<acceptance_criteria>

- HTTP POST to https://openrouter.ai/api/v1/chat/completions
- Authorization header uses OPENROUTER_API_KEY environment variable
- ModelResult returned with provider="openrouter"

- temperature=0 set in payload
- output_schema handling implemented with response_format
</acceptance_criteria>
</task>
