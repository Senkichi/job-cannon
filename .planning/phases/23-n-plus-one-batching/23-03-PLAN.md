<objective>
Move cancellation checks from per-job to pre-loop and replace per-job _update_session_counter calls with in-memory accumulators flushed once at batch end.

Purpose: Eliminate N+1 DB round-trips for cancellation polling and session counter updates in batch scoring background threads. For a 100-job batch, this removes ~200 unnecessary commits (100 cancellation SELECTs + 100 counter UPDATE+COMMITs).
Output: Modified batch_scoring.py with optimized background threads + new test_batch_scoring.py with verification tests.
</objective>
